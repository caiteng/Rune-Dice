# 开发路线

## 阶段 0：立项清理

目标：

- 清空旧工程实现。
- 写入新项目立项文档。
- 暂停旧 Web 部署工作流，避免误触旧脚本。

完成标准：

- 仓库不再保留旧游戏源码和旧构建产物。
- README 指向《猫猫守夜》。
- docs 中有项目、玩法和落地路线。

## 阶段 1：Godot 技术样板

目标：

- 使用 Godot 4.5 建立项目。
- 设置竖屏基准分辨率。
- 做出主场景、暂停层、UI 根节点。
- 实现浮动虚拟摇杆。
- 导出 Web 预览。

完成标准：

- Web 版可以在本地浏览器运行。
- 手机浏览器中下半屏摇杆可用。
- 项目结构适合后续抖音小游戏导出。

## 阶段 2：核心战斗循环

目标：

- 猫猫移动。
- 怪物从屏幕外生成并追踪。
- 自动攻击。
- 碰撞和伤害。
- 经验掉落和拾取。
- 升级三选一。

完成标准：

- 可以连续玩 3 分钟。
- 有死亡、暂停和重新开始。
- 基础节奏像幸存者类游戏。

## 阶段 3：武器和进化

目标：

- 实现 4 个优先武器。
- 实现 4 个对应饰品。
- 实现宝箱。
- 实现武器进化。

优先武器：

- 猫咪呼噜圈
- 毛线球环绕
- 激光笔
- 猫爪飞镖

完成标准：

- 每个武器可升级。
- 满级武器加对应饰品可以通过宝箱进化。
- 进化后表现和强度明显提升。

## 阶段 4：6 分钟 MVP

目标：

- 完整 6 分钟波次。
- 精英怪和最终 Boss。
- 结算界面。
- 小鱼干奖励。
- 局外基础升级。
- 广告 mock。

完成标准：

- Web 预览版可完整打一局。
- 用户试玩后能反馈怪物频率、武器强度、移动手感。

## 阶段 5：抖音小游戏适配

目标：

- 接入抖音 Godot SDK。
- 导出抖音小游戏。
- 抖音开发者工具运行。
- 真机预览。
- 广告接口接入适配层。

完成标准：

- 真机可以正常移动、战斗、升级和结算。
- 广告接口有失败兜底。
- 不依赖 Web 环境特性。

## 阶段 6：上线准备

目标：

- 完成基础信息、隐私政策、适龄提示等资料。
- 准备游戏截图、图标、简介。
- 检查广告合规。
- 提审版本。

完成标准：

- 完成第一次提审。
- 可以进入线上数据验证和后续内容扩展。
